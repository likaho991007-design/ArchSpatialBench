# Wild Data

Wild Data is the planned URL-only in-the-wild evaluation split.

It may include published architectural drawing sources such as ArchDaily project pages, but source image files are not redistributed.

Possible public materials:

- Source URLs
- Project metadata
- Checksums
- Benchmark annotations created by the project
- Benchmark questions
- Evaluation scripts

Wild Data results should be reported separately from Core Data results.

We will not scrape, bulk-download, or redistribute ArchDaily-hosted content without written permission.
